# Jiggle.js — Документация Фазы 3 (Component Model)

В третьей фазе фреймворк Jiggle.js получил полноценную поддержку компонентной модели с функциональными компонентами, пропсами и хуками жизненного цикла, как во Vue Composition API.

## 1. Функциональные компоненты

Компонент в Jiggle.js — это обычная JavaScript функция. Она может принимать объект `props`. 
Функция выполняет настройку компонента (создание реактивного состояния, подписку на события) и **возвращает `render`-функцию**, которая возвращает Virtual DOM (`h(...)`).

```javascript
function Counter(props) {
  const count = ref(props.initial || 0);

  // Возвращаем render-функцию (вызывается при каждом обновлении)
  return () => h('div', null,
    h('p', null, `Count: ${count.value}`),
    h('button', { onClick: () => count.value++ }, '+')
  );
}
```

## 2. Использование компонентов

Компоненты монтируются с помощью стандартной функции `h()`. Достаточно передать саму функцию-компонент первым аргументом:

```javascript
function App() {
  return () => h('div', null, 
    h('h1', null, 'My App'),
    h(Counter, { initial: 10 }) // Вставка компонента Counter
  );
}

render(h(App), '#container');
```

## 3. Реактивные Props

Свойства (props), передаваемые компоненту, автоматически оборачиваются в `reactive`. Если родительский компонент перерендерится с новыми пропсами, `instance.props` обновится, что заставит `render`-функцию дочернего компонента перерисоваться (если она обращается к изменённым пропсам).

## 4. Lifecycle Hooks (Хуки жизненного цикла)

Добавлены 3 основных хука жизненного цикла. Они должны вызываться синхронно внутри главной функции компонента (до возврата `render`-функции):

- **`onMounted(fn)`**: Вызывается после того, как компонент впервые отрендерился и был вставлен в реальный DOM.
- **`onUpdated(fn)`**: Вызывается после каждого реактивного обновления компонента (после отработки `patch`).
- **`onUnmounted(fn)`**: Вызывается прямо перед удалением компонента из DOM (например, если родительский компонент перестал его рендерить по условию).

```javascript
function Timer() {
  onMounted(() => console.log('Таймер запущен'));
  onUpdated(() => console.log('Таймер обновился'));
  onUnmounted(() => console.log('Таймер остановлен'));
  
  return () => h('div', null, 'Я таймер!');
}
```

## 5. Корректное размонтирование (`unmount`)

В ядро добавлена функция `unmount`. В отличие от старого вызова `removeChild`, новая функция `unmount` работает рекурсивно:
1. Запускает хук `onUnmounted` для удаляемого компонента.
2. Рекурсивно размонтирует все дочерние компоненты (чтобы у них тоже сработали хуки).
3. Только после этого физически удаляет DOM-узел.
