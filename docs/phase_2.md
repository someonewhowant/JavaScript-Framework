# Jiggle.js — Документация Фазы 2 (Virtual DOM)

В рамках второй фазы примитивный рендеринг через `innerHTML` был полностью заменён на реализацию **Virtual DOM** (VDOM). Это позволяет фреймворку обновлять только те элементы реального DOM, которые действительно изменились, что критически важно для производительности и сохранения состояния элементов (например, фокуса в `input`).

## 1. VNode и функция `h`

Функция `h(tag, props, ...children)` создаёт виртуальные узлы (VNode). VNode — это легковесный JavaScript-объект, описывающий часть DOM-дерева.

- **`tag`**: Строка с именем тега (например, `'div'`). Если `tag === null`, то это текстовый узел.
- **`props`**: Объект с атрибутами и слушателями событий (например, `{ class: 'app', onClick: () => ... }`).
- **`children`**: Массив дочерних VNode. Вложенные массивы сглаживаются, а строки и числа автоматически оборачиваются в текстовые VNode.

```javascript
const vnode = h('div', { class: 'card' },
  h('h1', null, 'Заголовок'),
  h('button', { onClick: handleClick }, 'Нажми меня')
);
```

## 2. Монтирование (`mount`)

Функция `mount(vnode, container)` принимает VNode и превращает его в реальные DOM-элементы, вставляя их в `container`. Она также сохраняет ссылку на реальный DOM-элемент внутри объекта VNode (`vnode.el = el`), что необходимо для последующего сравнения.

## 3. Алгоритм сравнения (`patch`)

Функция `patch(n1, n2)` (diff-алгоритм) принимает старый виртуальный узел (`n1`) и новый виртуальный узел (`n2`), сравнивает их и точечно обновляет реальный DOM:

1. **Сравнение тегов**: Если теги отличаются (например, `div` заменили на `span`), старый элемент полностью удаляется, а новый монтируется на его место.
2. **Текстовые узлы**: Если оба узла текстовые и их содержимое отличается, обновляется `textContent` существующего элемента.
3. **Обновление свойств (`props`)**: 
   - Новые/изменённые свойства (атрибуты и события) добавляются или перезаписываются.
   - Удалённые свойства удаляются из реального DOM.
4. **Обновление детей (`children`)**: Алгоритм рекурсивно вызывает `patch` для всех детей до общей длины массивов. Затем либо удаляет лишние элементы (если старых детей было больше), либо монтирует новые (если новых детей больше).

## 4. Обновлённая функция `render`

Функция `render(vnode, containerSelector)` теперь умная:
- При первом вызове она монтирует VNode в контейнер с помощью `mount`.
- При последующих вызовах (которые триггерятся системой реактивности из Фазы 1), она вызывает `patch`, передавая старый VNode (сохранённый в `container._vnode`) и новый VNode.

Благодаря этому переходу наше приложение в `app.js` больше не пересоздаёт весь HTML с нуля при каждом изменении `count` или `state.age`, а обновляет только конкретные текстовые узлы внутри соответствующих тегов `<p>`.
