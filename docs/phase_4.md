# Jiggle.js — Документация Фазы 4 (Шаблонизатор и Директивы)

В четвёртой фазе Jiggle.js обретает мощь декларативного написания интерфейсов. Больше не нужно вручную прописывать сотни вложенных функций `h()`. Теперь можно использовать обычный HTML-подобный синтаксис с директивами.

## 1. Функция `compile(template)`

Функция `compile` принимает строку с HTML-шаблоном и возвращает оптимизированную функцию рендеринга. 
Внутри под капотом происходит:
1. Парсинг HTML-строки в AST (Абстрактное синтаксическое дерево).
2. Анализ директив и атрибутов.
3. Кодогенерация (преобразование AST в код, вызывающий `h()`).
4. Оборачивание кода в `new Function` с использованием `with (ctx)`, что позволяет обращаться к реактивному состоянию напрямую из шаблона без дополнительных префиксов.

```javascript
const template = `<div class="card"><h1>{{ title }}</h1></div>`;
const renderFn = compile(template);

function App() {
  const title = ref('Hello, Compiler!');
  return () => renderFn({ title }); // Передаём контекст
}
```

## 2. Поддерживаемые директивы

Шаблонизатор Jiggle.js "из коробки" поддерживает 5 основных директив, перекрывающих 99% потребностей UI-разработки.

### `j-if` (Условный рендеринг)
Показывает или скрывает элемент (вместе с его дочерними узлами) в зависимости от истинности выражения.
```html
<p j-if="state.isVisible">Меня видно!</p>
```

### `j-for` (Рендеринг списков)
Создаёт список элементов на основе массива. Поддерживает синтаксис `item in items`.
```html
<ul>
  <li j-for="item in state.items">{{ item.name }}</li>
</ul>
```

### `j-model` (Двустороннее связывание)
Автоматически связывает значение `input` с реактивной переменной (ref или свойством объекта).
```html
<input j-model="searchQuery" placeholder="Поиск">
```

### `@event` / `j-on:event` (Обработка событий)
Слушает события DOM и вызывает указанную функцию.
```html
<button @click="increment">Увеличить</button>
<button j-on:mouseenter="hoverEffect">Наведи</button>
```

### `:attr` / `j-bind:attr` (Связывание атрибутов)
Позволяет динамически задавать значения атрибутам (например, `src`, `href`, `class`).
```html
<img :src="user.avatarUrl" :alt="user.name">
```

## 3. Автоматическая распаковка `ref` (Unwrapping)

Благодаря использованию `Proxy` на уровне контекста рендеринга, внутри шаблонов вам **не нужно** писать `.value` для переменных, созданных через `ref()`. Шаблонизатор и Jiggle.js сделают это за вас.

```javascript
const count = ref(10);
// В шаблоне достаточно написать:
// <p>{{ count }}</p> 
// И оно будет работать!
```
